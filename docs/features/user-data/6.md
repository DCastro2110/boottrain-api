# Subtask 6: Create user routes

## Description

Create `src/routes/user.route.ts` with GET and PUT endpoints.

## Details

1. Create `src/routes/user.route.ts` file
2. Import Fastify, ZodTypeProvider, Zod, auth helpers
3. Define Zod schemas:
   - Request body schema for PUT with all biometric fields (all optional)
   - Response schema for GET with all biometric fields
   - Error schemas (401, 403, 404, 500)
4. Create route configuration:
   - `PUT /users/{userId}`: Update user biometric data
     - Validate session
     - Check userId matches session user.id (Forbidden if not)
     - Call UpdateUserDataUseCase
     - Return 200 with userId
   - `GET /users/{userId}`: Retrieve user biometric data
     - Validate session
     - Check userId matches session user.id (Forbidden if not)
     - Call GetUserDataUseCase
     - Return 200 with user data
5. Export route plugin function

## Acceptance Criteria

- [ ] PUT /users/{userId} endpoint with proper Zod schema for request body
- [ ] GET /users/{userId} endpoint with proper Zod schema for response
- [ ] Both endpoints verify session and check userId authorization
- [ ] Proper error handling with correct HTTP status codes (401, 403, 404, 500)
- [ ] Follows existing route patterns from workout-plan.route.ts
- [ ] The code compiles without errors
