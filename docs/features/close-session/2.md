# Task 2: Create `CloseSessionUseCase` use case

## Description

Create the use case that contains all business logic for closing a session. Must validate ownership, existence, and session state (open vs closed).

## File to Create

`src/usecases/close-session-use-case.ts`

## Implementation Details

### Interface Definition

```typescript
interface InputDTO {
  userId: string;
  workoutPlanId: string;
  workoutDayId: string;
  sessionId: string;
}

interface OutputDTO {
  sessionId: string;
}
```

### Logic Flow

1. Fetch workout plan by `workoutPlanId`
2. If not found, throw `NotFoundError("Workout plan not found")`
3. If `workoutPlan.userId !== userId`, throw `ForbiddenError("You do not have permission to close this session")`
4. Find workout day in plan where `id === workoutDayId`
5. If not found, throw `NotFoundError("Workout day not found")`
6. Fetch session by `sessionId`
7. If not found, throw `NotFoundError("Session not found")`
8. If `session.completedAt !== null`, throw `BadRequestError("Session is already closed")`
9. Call `workoutSessionRepository.completeSession(sessionId)`
10. Return `{ sessionId: result.id }`

### Required Imports

- `NotFoundError`, `ForbiddenError`, `BadRequestError` from `../errors/errors.js`
- `IWorkoutPlanRepository` from `./create-workout-plan-use-case.js`
- `IWorkoutSessionRepository` from `./start-session-use-case.js`

## Acceptance Criteria

- [ ] Use case accepts `{ userId, workoutPlanId, workoutDayId, sessionId }` as input
- [ ] Throws `NotFoundError` if workout plan not found
- [ ] Throws `ForbiddenError` if user does not own the workout plan
- [ ] Throws `NotFoundError` if workout day not found in plan
- [ ] Throws `NotFoundError` if session not found
- [ ] Throws `BadRequestError` if session is already closed (`completedAt !== null`)
- [ ] Calls `workoutSessionRepository.completeSession()` on success
- [ ] Returns `{ sessionId: string }`
