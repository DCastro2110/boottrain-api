# Subtask 2: Create GetStats Use Case

## Description

Create `src/usecases/get-stats-use-case.ts` with input/output DTOs and business logic. This use case orchestrates the stats calculation using the existing `WorkoutSessionRepository`.

## Details

The use case should:

- Accept `userId`, `startDate`, and `endDate` as input
- Validate that `startDate` is before `endDate`
- Use `WorkoutSessionRepository.findByUserIdAndDateRange` to fetch sessions
- Calculate total sessions, total duration, and completion percent
- Return the stats in a structured output format

## Input/Output DTOs

```typescript
interface InputDTO {
  userId: string;
  startDate: Date;
  endDate: Date;
}

interface OutputDTO {
  totalSessions: number;
  totalDurationInSeconds: number;
  completionPercent: number;
  sessions: {
    sessionId: string;
    startedAt: Date;
    endedAt: Date | null;
  }[];
}
```

## Business Logic

- `totalSessions`: Count of all sessions in the date range
- `totalDurationInSeconds`: Sum of `(completedAt - startedAt)` for completed sessions only
- `completionPercent`: `(completed sessions / total sessions) * 100` (0 if no sessions)
- `sessions`: Array with `sessionId`, `startedAt`, `endedAt` for each session

## Error Handling

- If `startDate` >= `endDate`, throw an error with code `"BAD_REQUEST"`

## Acceptance Criteria

- File exists at `src/usecases/get-stats-use-case.ts`
- Exports `GetStatsUseCase` class
- InputDTO and OutputDTO interfaces defined
- Validates startDate is before endDate
- Calculates total sessions, total duration, completion percent
- Returns properly shaped OutputDTO
- Uses existing `WorkoutSessionRepository` to fetch data
