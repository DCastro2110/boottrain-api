# Subtask 2: Create Stats Repository Implementation

## Description

Create `src/db/stats-repository.ts` implementing `IStatsRepository` with Prisma queries to fetch workout sessions by user and date range.

## Details

The repository should:

- Import and implement `IStatsRepository` from `src/domain/stats.js`
- Use `prisma` client from `src/lib/db.js`
- Support optional transaction via `tx` parameter
- Map Prisma results to plain objects (convert `completedAt: null` properly)
- Order results by `startedAt` ascending

## Implementation Pattern

```typescript
import type { tx } from "../types/utils.js";
import { prisma } from "../lib/db.js";
import type {
  IStatsRepository,
  IWorkoutSessionForStats,
} from "../domain/stats.js";

export class StatsRepository implements IStatsRepository {
  private prismaClient = prisma;

  async findByUserIdAndDateRange(
    userId: string,
    startDate: Date,
    endDate: Date,
    tx?: tx,
  ): Promise<IWorkoutSessionForStats[]> {
    const client = tx ?? this.prismaClient;

    const sessions = await client.workoutSession.findMany({
      where: {
        userId,
        startedAt: {
          gte: startDate,
          lte: endDate,
        },
      },
      select: {
        id: true,
        startedAt: true,
        completedAt: true,
      },
      orderBy: {
        startedAt: "asc",
      },
    });

    return sessions.map((session) => ({
      id: session.id,
      startedAt: session.startedAt,
      completedAt: session.completedAt,
    }));
  }
}
```

## Acceptance Criteria

- File exists at `src/db/stats-repository.ts`
- Implements `IStatsRepository` interface
- Uses shared `tx` type for transaction support
- Returns properly mapped plain objects
- Results ordered by `startedAt` ascending
- Uses ESM imports with `.js` extension
- Repository instantiated with `private prismaClient = prisma` pattern
