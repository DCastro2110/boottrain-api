# Subtask 3: Create Stats Route

## Description

Create `src/routes/stats.route.ts` with GET `/stats` endpoint.

## Details

The route should:

- Accept `startDate` and `endDate` as query parameters (ISO 8601 date format: `YYYY-MM-DD`)
- Validate date format with Zod
- Check authentication via Better Auth
- Delegate to `GetStatsUseCase`
- Return stats response on success
- Handle errors appropriately

## Route Definition

```typescript
app.withTypeProvider<ZodTypeProvider>().route({
  method: "GET",
  url: "/stats",
  schema: {
    querystring: z.object({
      startDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
      endDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
    }),
    response: {
      200: z.object({
        totalSessions: z.number(),
        totalDurationInSeconds: z.number(),
        completionPercent: z.number(),
        sessions: z.array(
          z.object({
            sessionId: z.string().uuid(),
            startedAt: z.date(),
            endedAt: z.date().nullable(),
          }),
        ),
      }),
      400: ErrorSchema,
      401: ErrorSchema,
      500: ErrorSchema,
    },
  },
  handler: async (request, reply) => {
    // ...
  },
});
```

## Acceptance Criteria

- File exists at `src/routes/stats.route.ts`
- Route: `GET /stats?startDate=<>&endDate=<>`
- Query params validated with Zod (ISO 8601 format)
- Auth check returns 401 if no session
- Date validation (startDate < endDate) returns 400
- Returns 200 with stats on success
- Error handling for 400, 401, 500
- Uses ESM imports with `.js` extension
- Uses `fromNodeHeaders` from `better-auth/node` for auth
