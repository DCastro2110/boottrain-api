# Subtask 3: Create GetStats Use Case

## Description

Create `src/usecases/get-stats-use-case.ts` with `GetStatsUseCase` class containing business logic for calculating workout session statistics.

## Details

The use case should:

- Accept `userId`, `startDate`, and `endDate` as input
- Validate that `startDate` is before `endDate`
- Fetch sessions via repository
- Calculate:
  - `totalSessions`: Count of all sessions
  - `totalDurationInSeconds`: Sum of `(completedAt - startedAt)` for completed sessions only
  - `completionPercent`: `(completed sessions / total sessions) * 100`, or 0 if no sessions
- Return list of sessions with `sessionId`, `startedAt`, `endedAt`

## Input/Output DTOs

```typescript
interface GetStatsInputDTO {
  userId: string;
  startDate: Date;
  endDate: Date;
}

interface GetStatsOutputDTO {
  totalSessions: number;
  totalDurationInSeconds: number;
  completionPercent: number;
  sessions: Array<{
    sessionId: string;
    startedAt: Date;
    endedAt: Date | null;
  }>;
}
```

## Error Handling

- If `startDate >= endDate`, throw an error with code `"BAD_REQUEST"` and message `"Start date must be before end date"`

## Acceptance Criteria

- File exists at `src/usecases/get-stats-use-case.ts`
- Exports `GetStatsUseCase` class
- InputDTO and OutputDTO interfaces defined and exported
- Validates startDate is before endDate (throws error otherwise)
- Calculates total sessions correctly
- Calculates total duration only from completed sessions
- Calculates completion percentage (0 if no sessions)
- Returns properly shaped OutputDTO
- Uses ESM imports with `.js` extension
- Constructor accepts `IStatsRepository` interface
