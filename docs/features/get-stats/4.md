# Subtask 4: Create Stats Route

## Description

Create `src/routes/stats.route.ts` with `GET /stats` endpoint that accepts `startDate` and `endDate` query parameters.

## Details

The route should:

- Accept `startDate` and `endDate` as required query string parameters
- Validate date format using Zod
- Check authentication via Better Auth
- Delegate to `GetStatsUseCase`
- Return 200 with stats on success
- Return 401 if not authenticated
- Return 400 for invalid dates or date validation errors

## Route Schema

```typescript
const QuerystringSchema = z.object({
  startDate: z
    .string()
    .regex(/^\d{4}-\d{2}-\d{2}$/, "Invalid date format. Use YYYY-MM-DD"),
  endDate: z
    .string()
    .regex(/^\d{4}-\d{2}-\d{2}$/, "Invalid date format. Use YYYY-MM-DD"),
});

const ResponseSchema = z.object({
  totalSessions: z.number(),
  totalDurationInSeconds: z.number(),
  completionPercent: z.number(),
  sessions: z.array(
    z.object({
      sessionId: z.string().uuid(),
      startedAt: z.string().datetime(),
      endedAt: z.string().datetime().nullable(),
    }),
  ),
});
```

## Response Codes

| Code | Condition                                   |
| ---- | ------------------------------------------- |
| 200  | Success                                     |
| 400  | Invalid date format or startDate >= endDate |
| 401  | No session / unauthorized                   |

## Acceptance Criteria

- File exists at `src/routes/stats.route.ts`
- Route: `GET /stats?startDate=<>&endDate=<>`
- Query params validated with Zod schema
- Auth check returns 401 if no session
- Returns 200 with stats object on success
- Error responses include proper error codes (`"BAD_REQUEST"`, `"UNAUTHORIZED"`)
- Uses ESM imports with `.js` extension
- Uses `app.withTypeProvider<ZodTypeProvider>().route({...})` pattern
