# Subtask 1: Add findByUserIdAndDateRange to WorkoutSessionRepository

## Description

Add a new method `findByUserIdAndDateRange` to the existing `WorkoutSessionRepository` class that fetches all workout sessions for a user within a date range.

## Details

The method should:

- Accept `userId`, `startDate`, `endDate`, and optional `tx` parameters
- Query Prisma for workout sessions where `userId` matches and `startedAt` is within the date range (inclusive on both ends)
- Order results by `startedAt` ascending
- Map Prisma results to `IWorkoutSession[]` objects

## Method Signature

```typescript
async findByUserIdAndDateRange(
  userId: string,
  startDate: Date,
  endDate: Date,
  tx?: tx,
): Promise<IWorkoutSession[]>
```

## Implementation Pattern

```typescript
async findByUserIdAndDateRange(
  userId: string,
  startDate: Date,
  endDate: Date,
  tx?: tx,
): Promise<IWorkoutSession[]> {
  const client = tx ?? this.prismaClient;

  const sessions = await client.workoutSession.findMany({
    where: {
      userId,
      startedAt: {
        gte: startDate,
        lte: endDate,
      },
    },
    orderBy: {
      startedAt: "asc",
    },
  });

  return sessions.map((session) => ({
    id: session.id,
    userId: session.userId,
    workoutDayId: session.workoutDayId,
    startedAt: session.startedAt,
    completedAt: session.completedAt,
  }));
}
```

## Acceptance Criteria

- Method exists in `src/db/workout-session-repository.ts`
- Method signature matches: `findByUserIdAndDateRange(userId: string, startDate: Date, endDate: Date, tx?: tx): Promise<IWorkoutSession[]>`
- Sessions are filtered by `userId` and `startedAt` within range (inclusive)
- Results ordered by `startedAt` ascending
- Uses ESM imports with `.js` extension
- Uses shared `tx` type for transaction support
