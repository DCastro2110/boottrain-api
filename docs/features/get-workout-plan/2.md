# Task 2: Add `GET /workout-plans/:workoutPlanId` route

## Description

Add the GET route to `workout-plan.route.ts` following the existing pattern for start-session and close-session routes.

## File to Modify

`src/routes/workout-plan.route.ts`

## Route Specification

```
GET /workout-plans/:workoutPlanId
```

### Schema Definition

```typescript
params: z.object({
  workoutPlanId: z.string().uuid(),
}),
response: {
  200: z.object({
    id: z.string().uuid(),
    name: z.string(),
    description: z.string(),
    userId: z.string().uuid(),
    isActive: z.boolean(),
    workoutDays: z.array(
      z.object({
        id: z.string().uuid(),
        name: z.string(),
        isRestDay: z.boolean(),
        weekDay: weekDays,
        estimatedDurationInSeconds: z.number(),
        workoutExercises: z.array(
          z.object({
            id: z.string().uuid(),
            name: z.string(),
            restTimeInSeconds: z.number(),
            order: z.number(),
            sets: z.number(),
            reps: z.number(),
          }),
        ),
      }),
    ),
  }),
  401: ErrorSchema,
  403: ErrorSchema,
  404: ErrorSchema,
  500: ErrorSchema,
},
```

### Handler Logic

1. Extract `workoutPlanId` from `request.params`
2. Get auth session via `auth.api.getSession({ headers: fromNodeHeaders(request.headers) })`
3. If no session, return 401
4. Instantiate `WorkoutPlanRepository` and `GetWorkoutPlanUseCase`
5. Call `getWorkoutPlanUseCase.execute({ userId: session.user.id, workoutPlanId })`
6. On success, return 200 with workout plan data
7. Map errors:
   - `NOT_FOUND` -> 404
   - `FORBIDDEN` -> 403
   - else -> 500

## Acceptance Criteria

- [ ] Route registered with GET method at `/workout-plans/:workoutPlanId`
- [ ] UUID validation for `workoutPlanId` param
- [ ] 401 if no session
- [ ] 403 if user doesn't own the workout plan
- [ ] 404 if workout plan not found
- [ ] 200 with full workout plan data on success
- [ ] Follows existing route pattern from start-session and close-session
