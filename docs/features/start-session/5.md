# Subtask 5: Create POST sessions route

## Description

Add the POST sessions endpoint to `src/routes/workout-plan.route.ts`.

## Details

- File: `src/routes/workout-plan.route.ts`
- Add the route to the existing `workoutPlanRoutes` function
- Route path relative to prefix: `/:workoutPlanId/workout-days/:workoutDayId/sessions`

## Route Specification

```typescript
app.withTypeProvider<ZodTypeProvider>().route({
  method: "POST",
  url: "/:workoutPlanId/workout-days/:workoutDayId/sessions",
  schema: {
    params: z.object({
      workoutPlanId: z.string().uuid(),
      workoutDayId: z.string().uuid(),
    }),
    response: {
      201: z.object({
        sessionId: z.string().uuid(),
      }),
      400: ErrorSchema,
      401: ErrorSchema,
      403: ErrorSchema,
      404: ErrorSchema,
      409: ErrorSchema,
      500: ErrorSchema,
    },
  },
  handler: async (request, reply) => {
    // Implementation
  },
});
```

## Handler Implementation

1. Extract `workoutPlanId` and `workoutDayId` from `request.params`
2. Get session from Better Auth using `auth.api.getSession({ headers: fromNodeHeaders(request.headers) })`
3. If no session, return `401` with `{ error: "Unauthorized", code: "UNAUTHORIZED" }`
4. Create `WorkoutPlanRepository` and `WorkoutSessionRepository` instances
5. Create `StartSessionUseCase` with both repositories
6. Try to execute use case with `{ userId: session.user.id, workoutPlanId, workoutDayId }`
7. On success, return `201` with `{ sessionId: result.sessionId }`
8. Catch `NotFoundError`, `ForbiddenError`, `BadRequestError`, `ConflictError` and return appropriate status codes:
   - `NotFoundError` → 404
   - `ForbiddenError` → 403
   - `BadRequestError` → 400
   - `ConflictError` → 409
9. Catch all other errors, log and return `500` with `{ error: "Internal Server Error", code: "INTERNAL_SERVER_ERROR" }`

## Acceptance Criteria

- [ ] Route path: `POST /workout-plan/:workoutPlanId/workout-days/:workoutDayId/sessions`
- [ ] Path parameters validated as UUIDs with Zod
- [ ] Returns `201` with `{ sessionId }` on success
- [ ] Returns `400` if workout day is a rest day or workout plan is inactive
- [ ] Returns `401` if not authenticated
- [ ] Returns `403` if user doesn't own workout plan
- [ ] Returns `404` if workout plan or workout day not found
- [ ] Returns `409` if an active session already exists for this workout day
- [ ] Returns `500` on internal errors
