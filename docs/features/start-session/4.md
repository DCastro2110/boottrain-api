# Subtask 4: Create StartSessionUseCase

## Description

Create `src/usecases/start-session-use-case.ts` with business logic.

## Details

- File: `src/usecases/start-session-use-case.ts`
- Define `InputDTO`, `OutputDTO`, `IStartSessionUseCase` (interface), and the use case class
- Import types from domain and repository interface
- Import `NotFoundError`, `ForbiddenError`, `BadRequestError`, `ConflictError` from `../errors/errors.js`
- Import `WorkoutPlanRepository` from `../repositories/workout-plan-repository.js`

## Input/Output

```typescript
interface InputDTO {
  userId: string;
  workoutPlanId: string;
  workoutDayId: string;
}

interface OutputDTO {
  sessionId: string;
}
```

## Logic

1. Validate workout plan exists using `WorkoutPlanRepository.findById`
2. If not found, throw `NotFoundError` with message "Workout plan not found"
3. Validate workout plan is active (check `workoutPlan.isActive === true`)
4. If inactive, throw `BadRequestError` with message "Cannot start a session for an inactive workout plan"
5. Validate user owns the workout plan (check `workoutPlan.userId === input.userId`)
6. If not owned, throw `ForbiddenError` with message "You do not have permission to start a session for this workout plan"
7. Find the workout day in the plan (check `workoutPlan.workoutDays.find(d => d.id === input.workoutDayId)`)
8. If not found, throw `NotFoundError` with message "Workout day not found"
9. Validate workout day is not a rest day (check `workoutDay.isRestDay === false`)
10. If rest day, throw `BadRequestError` with message "Cannot start a session on a rest day"
11. Check if a session was already started today using `workoutSessionRepository.findByWorkoutDayIdAndStartedAt(input.workoutDayId, today)`
12. If a session exists with today's date, throw `ConflictError` with message "A session was already started today for this workout day"
13. Create session using `workoutSessionRepository.startSession(input.workoutDayId, input.userId)`
14. Return `{ sessionId: result.id }`
