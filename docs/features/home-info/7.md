# 7. Integrate route → use case → repository and harden error handling

## Description

Instantiate repository/use case in route handler and return computed output, with 500 fallback matching existing style.

## Acceptance Criteria

- Route uses Better Auth session retrieval and passes `session.user.id` into use case.
- Successful response returns `200` with exact output contract.
- Error path logs server error and returns `500` with standard error body.
- No untyped `any` leaks in route/use case/repository integration.
