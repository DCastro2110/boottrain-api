# Task 1: Add `timezone` field to Better Auth session

## Description

Configure Better Auth to capture and store the user's timezone in the session. This requires modifying the auth configuration to include a custom session field.

## File to Modify

`src/lib/auth.ts`

## Implementation Details

Better Auth allows customizing the session object. You need to:

1. Configure the session to include a `timezone` field
2. Use Better Auth MCP to get the user's timezone when the session is created

The session customization in Better Auth typically uses the `session` option with additional fields. However, since Better Auth's session customization may vary, use the Better Auth MCP tool to understand how to properly add custom fields to the session.

### Key Points

- Session customization happens in the `betterAuth()` configuration
- Custom fields can be added to track additional session data
- The timezone should be captured when the session is initiated

## Acceptance Criteria

- [ ] Session includes `timezone` field
- [ ] Timezone is captured when session is started
- [ ] Session type is updated to reflect the new field
