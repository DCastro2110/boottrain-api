# Task 3: Create `GetWorkoutDayUseCase` use case

## Description

Create the use case that fetches a workout day by ID, validates ownership, and looks up open sessions.

## File to Create

`src/usecases/get-workout-day-use-case.ts`

## Implementation Details

### Interface Definition

```typescript
interface InputDTO {
  userId: string;
  workoutPlanId: string;
  workoutDayId: string;
}

interface OutputDTO {
  weekDay: string;
  name: string;
  estimatedDurationInSeconds: number;
  numberOfExercises: number;
  coverImageUrl: string | null;
  workoutSessionId: string | null;
  workoutExercises: Array<{
    name: string;
    reps: number;
    sets: number;
    description: string;
    estimatedDurationInSeconds: number;
  }>;
}
```

### Logic Flow

1. Accept `{ userId, workoutPlanId, workoutDayId }` as input
2. Call `workoutPlanRepository.findWorkoutDayById(workoutPlanId, workoutDayId)`
3. If not found, throw `NotFoundError("Workout day not found")`
4. If found but `userId` does not match, throw `ForbiddenError("You do not have permission to access this workout day")`
5. Check for open session in current week using `WorkoutSessionRepository.findOpenSessionInCurrentWeek(workoutDayId)`
6. Return workout day data with `workoutSessionId` (null if no open session)

### Open Session Lookup

Use `dayjs` to determine if there's an open session started in the current week:

1. Get current date
2. Calculate start of current week (e.g., dayjs().startOf('week'))
3. Calculate end of current week (e.g., dayjs().endOf('week'))
4. Query for sessions with `startedAt` within this range and `completedAt` is null

### Required Imports

- `NotFoundError`, `ForbiddenError` from `../errors/errors.js`
- `IWorkoutPlanRepository` from `./create-workout-plan-use-case.js`
- `IWorkoutSessionRepository` from `./start-session-use-case.js`
- `dayjs` for date handling
