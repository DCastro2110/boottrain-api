# Task 2: Create `findWorkoutDayById` repository method

## Description

Add a method to `WorkoutPlanRepository` to fetch a specific workout day with its exercises.

## File to Modify

`src/repositories/workout-plan-repository.ts`

## Implementation Details

Add a new method `findWorkoutDayById(workoutPlanId, workoutDayId)` that:

1. Finds the workout plan by ID (returns null if not found)
2. Finds the workout day within that plan (returns null if not found)
3. Returns workout day data with exercises and computed `numberOfExercises`

### Return Type

```typescript
{
  id: string;
  name: string;
  weekDay: weekDays;
  estimatedDurationInSeconds: number;
  numberOfExercises: number;
  coverImageUrl: string | null;
  workoutExercises: Array<{
    name: string;
    reps: number;
    sets: number;
    description: string; // Note: description field not in schema, may need to add or omit
    estimatedDurationInSeconds: number;
  }>;
}
```

**Note:** The schema does not include `description` or `estimatedDurationInSeconds` on `WorkoutExercise`. Verify what fields exist and adjust the return type accordingly. If `description` doesn't exist in the schema, it may need to be added, or the requirement may have been based on a different schema version.

### Required Interface Update

The interface `IWorkoutPlanRepository` in `create-workout-plan-use-case.ts` will need a new method signature.

## Acceptance Criteria

- [ ] Method accepts `workoutPlanId` and `workoutDayId` as parameters
- [ ] Returns workout day with exercises or null
- [ ] Includes `numberOfExercises` in the return type (computed from workoutExercises length)
- [ ] Handles non-existent workout plan and workout day gracefully
- [ ] Repository implements the new interface method
